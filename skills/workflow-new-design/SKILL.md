name: workflow-new-design
description: "Create a new feature design through iterative discussion. Activates when a user wants to design something new, e.g. '我要设计一个 xxx 功能', '我想实现 xxx', 'design a feature', 'new design'. Outputs a human-readable DESCRIPTION.md that serves as a shared contract between human and agent, later refined into PLAN.md by workflow-plan-design and implemented by workflow-implement-design."
---

## Instructions

1. If `$ARGUMENTS` is provided, treat it as the initial idea or topic. Otherwise, ask the user what feature they want to design. Use Chinese for conversation.

2. Interview the user about their idea until reaching a shared understanding. Cover these areas in order, resolving each before moving on:
   - What problem does this feature solve, and for whom?
   - What are the expected inputs, outputs, or user interactions?
   - What constraints or non-goals should be respected?
   - Are there dependencies on other features or systems?
   - Are there edge cases or failure modes to address?

   If a question can be answered by exploring the codebase, do so instead of asking. Stop when all areas above are resolved and the user confirms the design is complete.

3. Once the design is fully discussed and agreed upon, do the following:
   - Suggest a slug name (English, hyphen-separated) based on the feature content; confirm with the user before proceeding
   - Get today's date in YYYYMMDD format
   - If `designs/` does not exist, create it first. Then create `designs/YYYYMMDD-slug/` inside it (same directory used by workflow-plan-design and workflow-implement-design)
   - If a design with the same slug already exists, create a new version instead of overwriting (e.g., `designs/YYYYMMDD-slug-v2/`)
   - Write `DESCRIPTION.md` in Chinese (human-readable contract), containing at minimum:
     - **功能描述**: What the feature does and why
     - **用户故事**: User stories in "作为...我希望..." format
     - Additional sections as appropriate (e.g., 设计细节, 约束条件, 参考引用)
     - If the design references other designs, include references naturally in the relevant sections
   - DESCRIPTION.md is the human-facing contract. A separate PLAN.md (English, agent-facing) will be generated by workflow-plan-design.

4. After writing DESCRIPTION.md, ask the user to confirm the following before proceeding:
   - Does the 功能描述 accurately reflect the discussed requirements?
   - Do the 用户故事 cover the main scenarios?
   - Are constraints and boundaries clearly stated?

   Only after confirmation, inform the user that the next step is to run `/workflow-plan-design` to generate the implementation plan. The full workflow chain is:
   - workflow-new-design → creates `designs/YYYYMMDD-slug/DESCRIPTION.md` (Chinese, human contract)
   - workflow-plan-design → reads DESCRIPTION.md → creates `designs/YYYYMMDD-slug/PLAN.md` (English, agent contract)
   - workflow-implement-design → reads PLAN.md → executes step by step
   - All three workflows operate in the same directory.
