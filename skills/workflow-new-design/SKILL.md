---
name: workflow-new-design
description: "Create a new feature design through iterative discussion. Activates when a user wants to design something new, e.g. '我要设计一个 xxx 功能', '我想实现 xxx', 'design a feature', 'new design'. Outputs a human-readable DESCRIPTION.md that serves as a shared contract between human and agent, later refined into PLAN.md by workflow-plan-design and implemented by workflow-implement-design."
---

## Instructions

1. If `$ARGUMENTS` is provided, treat it as the initial idea or topic. Otherwise, ask the user what feature they want to design. Use Chinese for conversation; the skill itself operates in English.

2. Interview the user relentlessly about every aspect of their idea until reaching a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one-by-one. If a question can be answered by exploring the codebase, explore the codebase instead of asking.

3. Once the design is fully discussed and agreed upon, do the following:
   - Suggest a slug name (English, hyphen-separated) based on the feature content
   - Get today's date in YYYYMMDD format
   - Create the directory `designs/YYYYMMDD-slug/` (same directory used by workflow-plan-design and workflow-implement-design)
   - If a design with the same slug already exists, create a new version instead of overwriting (e.g., `designs/YYYYMMDD-slug-v2/`)
   - Write `DESCRIPTION.md` in Chinese (human-readable contract), containing at minimum:
     - **功能描述**: What the feature does and why
     - **用户故事**: User stories in "作为...我希望..." format
     - Additional sections as appropriate (e.g., 设计细节, 约束条件, 参考引用)
     - If the design references other designs, include references naturally in the relevant sections
   - DESCRIPTION.md is the human-facing contract. A separate PLAN.md (English, agent-facing) will be generated by workflow-plan-design.

4. After writing DESCRIPTION.md, inform the user that the next step is to run `/workflow-plan-design` to generate the implementation plan. The full workflow chain is:
   - workflow-new-design → creates `designs/YYYYMMDD-slug/DESCRIPTION.md` (Chinese, human contract)
   - workflow-plan-design → reads DESCRIPTION.md → creates `designs/YYYYMMDD-slug/PLAN.md` (English, agent contract)
   - workflow-implement-design → reads PLAN.md → executes step by step
   - All three workflows operate in the same directory.
